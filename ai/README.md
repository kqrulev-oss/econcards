# EconCards — AI-тренер (бесплатный прокси)

AI работает через маленький прокси на **Cloudflare Worker**, который держит
ключ **Gemini** на сервере (в браузер он не попадает). На бесплатных тирах
Cloudflare + Google AI Studio это **$0** в эксплуатации на небольшой аудитории.

Приложение остаётся статикой на GitHub Pages; к Worker'у ходят только
AI-запросы, и при отсутствии сети/URL функции просто прячутся.

## Шаг 1. Бесплатный ключ Gemini (1 минута)
1. Открой https://aistudio.google.com/apikey (нужен только Google-аккаунт, карта не требуется).
2. «Create API key» → скопируй ключ.

## Шаг 2. Развернуть Worker
Вариант через веб-интерфейс (без установки):
1. Зайди на https://dash.cloudflare.com → **Workers & Pages** → **Create** → **Worker**.
2. Назови, например, `econcards-ai`, нажми **Deploy**, затем **Edit code**.
3. Вставь содержимое `econcards-ai-worker.js`, снова **Deploy**.
4. **Settings → Variables and Secrets**:
   - добавь секрет `GEMINI_KEY` = твой ключ из шага 1;
   - (необязательно) `MODEL` — если модель по умолчанию `gemini-2.0-flash` выдаёт ошибку, поставь актуальную из AI Studio;
   - (необязательно) `DAILY_LIMIT` — лимит запросов на IP в сутки (по умолчанию 40).

### (Необязательно) дневной лимит
Чтобы ограничить расход и защититься от абьюза:
1. **Storage & Databases → KV → Create namespace** (например `ECONCARDS_RATE`).
2. В Worker: **Settings → Bindings → Add → KV namespace**, имя переменной `RATE`, выбери namespace.

Локально то же самое через Wrangler:
```
npx wrangler deploy econcards-ai-worker.js --name econcards-ai
npx wrangler secret put GEMINI_KEY
```

## Шаг 3. Подключить в приложении
Скопируй URL Worker'а (вида `https://econcards-ai.<твой>.workers.dev`) и вставь
его в приложении: вкладка **«Я» → AI-тренер → Адрес AI-сервера**. Всё —
кнопки «Проверить решение», «Подсказка», «Объясни проще», «Похожая задача»
появятся на задачах с разбором.

## Что уходит на сервер
Текст условия, эталонного решения и (при проверке) решение ученика. Промпты
строятся на сервере, ключ клиенту не отдаётся. Сгенерированные «похожие
задачи» помечаются как непроверенные и не попадают в основную базу.
